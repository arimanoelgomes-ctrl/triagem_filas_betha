# Documentação Específica — Vertical Arrecadação

Esta pasta armazena documentação **específica do domínio Arrecadação/ISS** (Tributos, Procuradoria, Cidadão Web, Livro Eletrônico, e-Nota, Protocolo). Para documentação compartilhada com outras verticais (incidentes da plataforma, padrões gerais), veja `../../docs/`.

## Conteúdo sugerido

- **Glossário técnico** — vocabulário específico (IPTU, ITBI, ISS, dívida ativa, CDA, e-Sfinge, SINTER, CIB, marco prescricional etc.).
- **Exemplos de chamados** — exemplos anonimizados de chamados-padrão e seus comentários internos esperados.
- **Regras de negócio** — anotações sobre LC 116/2003, código tributário municipal, alíquotas de ISS, taxas, alíquotas de IPTU, simples nacional, MEI etc.
- **Mapeamento de produtos** — relação entre os produtos (Tributos, Procuradoria, Gestão Fiscal, e-Nota, Cidadão Web, Livro Eletrônico, Protocolo) e seus principais módulos/erros conhecidos.
- **e-Sfinge / TCE-SC** — referência dos scripts `[Tributos] e-Sfinge Online 2026 - 01 a 12`, validações de assuntos (saldo inicial, lançamentos, dívida ativa, CONs), erros recorrentes (BTH-4017, BTH-4028) e tratamentos via script BSL.
- **Convênios bancários** — Sicoob, Sicredi, BB, CEF, Bradesco, Itaú: tokens, layouts (Febraban × registro), troubleshooting de baixa.
- **Decisões de projeto** — registros (ADRs) sobre mudanças de comportamento da triagem ao longo do tempo, específicas desta vertical.

> Esta pasta é versionada no Git e seu conteúdo serve como fonte de consulta para a IA e para os analistas.
